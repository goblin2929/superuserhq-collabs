01 — Products

Three Ways to Use Claude

Claude comes in three product forms, each with a different level of autonomy. Think of it as a spectrum: from conversation to delegation to full independence.

App

Claude App

The chat interface at claude.ai. You ask, it answers. Good for quick questions, brainstorming, and one-off tasks.

Talk — you ask, it answers
Cowork

Claude Cowork

An agentic assistant that can use tools, browse the web, write files, and execute multi-step tasks in a shared workspace.

Delegate — hand off tasks, it executes
Code

Claude Code

A terminal-based autonomous agent. It reads your codebase, runs commands, builds entire workflows, and operates independently.

Open a studio — let go, it works autonomously
You're already here — your team has been using Cowork for 3 weeks.
Today we go further. Let's unlock what Code can do.

04 — Why Code

Why Claude Code

Two real differences set Claude Code apart from Cowork. They aren't about speed — they're about what becomes possible.

Connect to Anything

Internal APIs, databases, third-party services, CLIs, SDKs. If it has an interface, Code can talk to it. Cowork is limited to pre-built integrations.

Team Governance

Version-controlled instructions, multi-layer permissions, audit trails, team-wide standards. Enterprise-grade guardrails that Cowork doesn't have.

# Step Cowork Code Why
1 Write video script from brief ✓ ✓ Either
2 Generate scenes via Higgsfield API ✕ ✓ API
3 Stitch clips with CapCut CLI ✕ ✓ CLI
4 Write caption & hashtags ✓ ✓ Either
5 Upload to platform via API ✕ ✓ API
3 out of 5 steps are Code-only.

05 — Concepts

Core Concepts

Four building blocks that make Claude Code powerful. Master these and you can build almost anything.

MCP (Model Context Protocol)

Universal connectors that let Claude talk to external tools — APIs, databases, SaaS products. Think USB ports for AI.

// .mcp.json { "mcpServers": { "google-sheets": { "command": "npx", "args": ["@anthropic/sheets-mcp"] } } }
USB ports for your AI — plug in any tool

Skills

Reusable instruction sets that teach Claude how to do specific tasks — like SOPs for your AI assistant. Triggered with /skill-name.

# ~/.claude/skills/write-blog.md # Blog Writer Skill ## Steps 1. Read brand voice from CLAUDE.md 2. Research topic using web search 3. Write draft following template 4. Apply SEO best practices
A recipe book — each skill is a step-by-step recipe

CLAUDE.md

A configuration file that tells Claude who it is, what standards to follow, and how to behave. Lives at the root of your project.

# CLAUDE.md # Brand Voice - Tone: professional, warm - Audience: B2B travel executives - Never use: "revolutionary", "synergy" - Always include: data + proof points
An employee handbook — standards everyone follows

MEMORY.md

Persistent memory across sessions. Claude remembers past decisions, learnings, and context so it doesn't start from scratch every time.

# MEMORY.md ## Learnings - Blog posts > 2000 words rank better - VP prefers bullet-point summaries - Campaign naming: YYYYMM-channel-topic ## Decisions - Approved: new content calendar format - Rejected: automated social posting
A journal — it remembers so you don't have to repeat yourself
06 — Architecture

Team Architecture

The difference between Cowork and Code isn't just features — it's governance. Here's how each structures team instructions.

Cowork

Flat 2-Layer Structure

Each person has their own Projects. No shared layer. No inheritance. No central control.

Person A — their projects
Person B — their projects
Person C — their projects

No connection between team members.

Code

Multi-Layer with Inheritance

Layered system where org standards flow down to every team member automatically via Git.

Organization CLAUDE.md
↓
Project CLAUDE.md
↓
Personal .claude/ overrides

Standards cascade. Everyone stays aligned.

Governance Gaps in Cowork

No Version Control

Instructions change without history. No way to audit what changed or roll back mistakes.

No Shared Standards

Each person configures Claude differently. Brand voice, tone, and quality vary across team.

No Permission Layers

No distinction between org-level rules and personal preferences. Everything is flat.

No Audit Trail

VPs and managers can't review what instructions agents are following or verify compliance.

07 — Getting Started

Getting Started with Claude Code

Six steps from zero to your first team workflow. Follow along — this is the hands-on part.

1

Open Terminal

Press Cmd+Space, type "Terminal", hit Enter. This is your command center.

$ # Cmd+Space → "Terminal" → Enter
2

Navigate to Your Project Folder

Use cd to move into the directory where your project lives. This is where Claude will operate.

$ cd ~/my-project
3

Launch Claude Code

Type claude and press Enter. Claude boots up and reads your project context automatically.

$ claude ╭──────────────────────────────────╮ │ Claude Code v1.x │ │ Reading project context... │ │ Ready. │ ╰──────────────────────────────────╯
4

Create Your CLAUDE.md

Define your team's brand voice, quality standards, and guardrails. This file is the foundation of everything.

$ cat > CLAUDE.md << 'EOF' # Team Standards ## Brand Voice - Professional, data-driven, warm - Audience: B2B travel executives - Language: English & Bahasa Indonesia ## Quality Gates - All content reviewed before publishing - Include data sources for claims EOF
5

Build Your First Skill

Create a reusable skill file. This teaches Claude a repeatable workflow your team uses often.

$ mkdir -p .claude/skills $ cat > .claude/skills/weekly-report.md << 'EOF' # Weekly Report Skill ## Goal Generate a weekly marketing performance summary. ## Steps 1. Pull data from Google Analytics 2. Summarize key metrics vs last week 3. Highlight top-performing content 4. Flag items needing attention 5. Format as bullet-point brief for VP EOF
6

Share with Your Team

Commit and push. Now every team member who pulls the repo gets the same standards and skills.

$ git add CLAUDE.md .claude/ $ git commit -m "Add team standards and first skill" $ git push origin main
08 — Skills

Building Great Skills

Skills are the building blocks of team AI. A well-written skill turns a vague request into consistent, high-quality output every time.

Anatomy of a Skill File

# skill-name.md ## Goal One sentence: what this skill produces. ## Context Who uses this, when, and why. ## Inputs - What the user provides (topic, data, brief) - What Claude reads from context (CLAUDE.md, files) ## Steps 1. First, do this... 2. Then, do this... 3. Finally, produce this output. ## Output Format Describe the expected deliverable format. ## Quality Checks - [ ] Matches brand voice - [ ] Includes data citations - [ ] Under 500 words

Be Specific

Write clear, concrete instructions. "Write a 300-word summary" beats "write a summary." Specificity reduces variance.

Include Examples

Show Claude what "good" looks like. Include sample outputs, tone examples, or reference docs. One example is worth a hundred instructions.

Iterate Relentlessly

Run the skill, review the output, fix the instructions. Good skills take 3-5 iterations to get right. Don't stop at version one.

09 — Advanced

Advanced Features

Once you're comfortable with the basics, these features let you build truly powerful agent systems.

Hooks

Run scripts before or after Claude actions. Enforce code formatting, linting, or custom validation automatically.

Subagents

Spawn child agents for parallel tasks. One agent researches while another writes. Divide and conquer complex workflows.

Agent Teams

Define named agents with specific roles — writer, researcher, analyst, reviewer. Orchestrate them from a lead agent.

Plan Mode

Ask Claude to plan before executing. Review and approve the plan, then let it run. More control for high-stakes tasks.

MCP Plugins

Install pre-built connectors from the community. Google Sheets, HubSpot, Airtable, Slack — hundreds available and growing.

Skill Creator

Use /skill-creator to generate new skills interactively. Describe what you need and Claude builds the skill file for you.

Evals

Measure how well your skills and prompts actually perform. Run automated test cases, compare outputs across iterations, and catch regressions before they reach your team.

Git Worktrees

Work on multiple branches simultaneously without stashing or switching. Each worktree is a separate folder with its own branch — like having multiple desks for the same project.

Managed Agents

Anthropic’s hosted agent infrastructure. Define an agent (model, tools, MCP servers), spin up a cloud container, and let Claude run autonomously for minutes or hours — no custom loop or sandbox needed.

10 — Philosophy

The Mindset Shift

Using Claude Code well isn't about prompting tricks. It's about thinking differently about how you work with AI.

01

Goals, Not Steps

Tell Claude what you want to achieve, not how to do it. "Increase blog traffic 20%" beats "Write 5 blog posts about X." Let the agent figure out the path.

02

Chain, Don't Repeat

Build on previous outputs. Use skills that reference other skills. Create pipelines where one step feeds the next, instead of starting from scratch each time.

03

Document Your Workflows

If you do something twice, write a skill. If the skill works, share it with the team. The best teams turn tacit knowledge into reusable instructions.

04

Collect References

Feed Claude examples of great work — competitor campaigns, top-performing content, style guides. The more context it has, the better the output.

05

Iterate Relentlessly

Version one is never the final version. Review outputs, refine instructions, test again. The teams that iterate fastest win — and AI makes iteration nearly free.
