Two modes
Deterministic workflows vs. agentic workflows.
Recipe vs. chef. Both belong in a real kitchen.
Deterministic = n8n workflows  ·  Agentic = Skills & Agents (Claude Code)
n8n workflows · deterministicSkills & Agents · agentic
How it worksFollows the same steps every timeLooks, thinks, decides, adjusts
Where it winsHigh-volume, repeatable, auditable work — 100% consistency per step. A 10-step chain of 95%-accurate agents = only ~60% end-to-end.Judgment-heavy, messy, novel-input work.
AdaptsRigid — reshaping the flow means rebuilding it.Bends with you — today's one-off becomes tomorrow's reusable skill.
RisksTechnical to build — wiring nodes takes real setup; brittle when inputs drift.Non-deterministic — same prompt, different answer. The craft is keeping it consistent and correct.

Section 216
Three concepts from that story
Agent. Hook. Gate.
Agent

Isolated specialist.

Fresh context window. Specific tools. The collect agent never sees the report agent's conversation. Each gets a clean whiteboard.

Hook

Harness-level enforcement.

Your system runs it, not Claude. A tripwire. PreToolUse blocks writes to .env. PostToolUse re-verifies every edit. Stop extracts lessons.

Gate

Deterministic validation.

Python, not prompts. "Report says 27,659 keywords. Raw file says 15,164. Don't match → exit 1 → retry." The model cannot explain away an exit code.

Monolith
→ Agents gave isolation
Drowning rules
→ Hooks gave enforcement
Fake numbers
→ Gates gave validation
Agents give you isolation. Hooks give you enforcement. Gates give you validation. Together: the harness.
