Section 1 · The building blocks07a
Skill
Skill. A reusable procedure Claude auto-loads.
What it is
A markdown file in ~/.claude/skills/ with a name, a description, and a body of steps.
On session start, Claude reads only the 1-line description — the body loads only when invoked. Cheap on context.
How to activate
Just describe what you want. If the skill's description matches, Claude loads it automatically.
Or force-invoke with a slash command: /blog-production.
When to use it
A procedure you'll repeat — same steps, different inputs.
"Run the affiliate performance report for the current month." "Write a blog brief." "Format a monthly report." Codify the SOP once — run it forever.
skills/blog-production.md
---
name: blog-production
description: Use when writing blog content — runs the 7-phase production workflow.
---
## Steps
1. Research · 2. Outline · 3. Draft · …
The description is the most important line. That's how Claude finds it.

Section 1 · The building blocks07c
What makes a good skill
What makes a good skill.
Skills are easy to write. Easy to write badly. Five rules separate the two.
TraitGoodBad
01 · Sharp description "Use when writing a monthly affiliate performance report — runs pull → analysis → deck." "Affiliate tool." ← Claude never finds it.
02 · One job One skill = one procedure. Splits naturally by verb. "Do-everything marketing skill." ← nothing triggers cleanly.
03 · Names its inputs "Requires: GA4 property ID, month, affiliate list." Assumes context. ← fails silently on Day 2.
04 · Short body, linked references Steps + a link to references/report-template.md. Loads the ref only if needed. 2,000 lines pasted in the skill body. ← bloats every load.
05 · Testable output "Output: a Google Sheet with columns X / Y / Z + a summary block." "Gives you insights." ← you can't tell if it's wrong.
The description is the trigger. The body is the recipe. References are the pantry — load only when cooking.
A good skill you wrote once → your team runs forever.

What it is

Skill vs. Agent
Skill vs. Agent.
Two different tools. Knowing when to reach for which is the whole game.
SkillAgent
Mental modelA recipe cardA specialist chef
ScopeOne repeatable procedureA role with judgment + tools
Context costJust the description (cheap)Runs on its own whiteboard (free to your context)

Section 2 · Context window14i
Playbook · Level 3 — architecture
The playbook. Level 3 — architecture.
When the same problem keeps recurring — redesign the system, not the prompt.
01 · Delegate to subagents
Task needs 5+ files or 10+ pages? Spawn an Agent (Explore / Plan / general-purpose). Subagent has its own whiteboard — returns a summary. Your context stays clean.
02 · Skills over inlined procedures
Procedure 20+ lines and not needed every session? Make a skill. The harness loads only the 1-line description at startup. Full body loads only when invoked.
03 · Extract, don't dump
Reading a 200-page PDF = ~400K tokens. Grep/Glob first, then Read with offset + limit. For MCP — filter before retrieval (date ranges, specific pages).
04 · Hooks for zero-context enforcement
Rule that must fire — don't rely on Claude's judgment. Add a PostToolUse hook in settings.json. Runs in shell, not model context. Injects a reminder only when the event fires.
Most issues fix with Levels 1–2. Level 3 is when the system itself is the problem.
Whiteboard · Cabinet · Desk · Wall — design against the right surface.

Section 216
Three concepts from that story
Agent. Hook. Gate.
Agent

Isolated specialist.

Fresh context window. Specific tools. The collect agent never sees the report agent's conversation. Each gets a clean whiteboard.

Hook

Harness-level enforcement.

Your system runs it, not Claude. A tripwire. PreToolUse blocks writes to .env. PostToolUse re-verifies every edit. Stop extracts lessons.

Gate

Deterministic validation.

Python, not prompts. "Report says 27,659 keywords. Raw file says 15,164. Don't match → exit 1 → retry." The model cannot explain away an exit code.

Monolith
→ Agents gave isolation
Drowning rules
→ Hooks gave enforcement
Fake numbers
→ Gates gave validation
Agents give you isolation. Hooks give you enforcement. Gates give you validation. Together: the harness.
